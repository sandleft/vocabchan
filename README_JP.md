# VocabChan 📸

[English](./README.md) · [中文](./README_CN.md) · [日本語](./README_JP.md)

> ホットキーを押すだけ。AIが解析して、ノートに自動保存。それだけ。

> ⚠️ **Windows版EXEは現在ベータテスト中です。バグが出る可能性があるため、今はPythonソースからの実行を推奨します。**

---

## これは何？

VocabChanは「見ながら学ぶ」人のための語学学習ツールです。

アニメ、ドラマ、ゲーム、YouTubeを見ていてわからない表現が出てきたら、ホットキーを一つ押すだけ。スクリーンショット・録音・クリップボードのテキストを、学習言語に特化したプロンプトと一緒にAIに送ります。返ってくるのは単なる翻訳ではなく、その言語の学習者が本当につまずく文法ポイントや語法の深い解説です。結果はObsidianのノートに自動保存、必要ならAnkiカードにも同期。ウィンドウを切り替える必要も、コピペする必要もありません。

---

## できること

**何でもキャプチャ**
- 画面をスクリーンショットしてAIに解説してもらう
- マイクまたはシステム音声を録音して自動文字起こし＋解析
- OBSの直近N秒の映像を取得してAIに解析させる
- クリップボードのテキストをワンキーで解析
- 単語リストをまとめて貼り付けて一括処理
- キャプチャ範囲を選択して画面の一部だけ送る
- 送信前にプレビュー確認ウィンドウを表示するオプションあり

**AI解析**
- 選択中の言語テンプレートに合わせた専用プロンプトで、設定した任意のAIプロバイダーに送信（詳細は下の言語テンプレートセクション参照）
- 返ってくる内容：翻訳、単語詳細、文法の深掘り、使用場面、発音メモ
- 最近学んだ単語を毎回プロンプトに注入してAIが関連を見つけやすくする（RAGメモリ）
- 重複検出：同じ単語が何度も出てきたら何回目かを通知
- APIリクエストが失敗したら自動リトライキューに入る、記録は消えない
- ローカルOCR（PaddleOCR）でAI送信前にテキスト抽出、幻覚を減らせる（任意）
- ローカル音声文字起こし（Faster-Whisper）でAPI不要でオフライン動作（任意）
- 常駐バックグラウンドリスナー：キーを押さなくても自動トリガー *(CPU負荷が非常に高いため注意)*

**全部保存**
- 毎回の記録がObsidianに構造化ノートとして保存される（単語・原文・解析・画像・音声・動画すべて一つのノートに）
- Obsidianの[Spaced Repetitionプラグイン](https://github.com/st3v3nmw/obsidian-spaced-repetition)と組み合わせればObsidian内だけで間隔反復復習が完結する
- 単語エントリーが自動でObsidianナレッジグラフのノードになり、単語同士のつながりが可視化される
- Ankiにフラッシュカードとして同期も可能（画像・音声対応）
- **AnkiはビデオをサポートしていないためビデオキャプチャはObsidianにのみ保存される**
- Ankiカードの表面内容は設定可能：単語のみ・文のみ・両方の3択
- Ankiの音声再生タイミングは設定可能：カードを裏返す前または後
- メディアファイルをTeldriveにバックアップする機能もあり

**復習と統計**
- 3日・7日・14日後に復習リマインダーを自動で出す（アプリ内蔵）
- 保存済み単語の検索
- 学習統計：総単語数、今日、今週、言語別、高頻度単語
- 単語リストをCSVまたはTXTにエクスポート

**おまけ**
- 毎晩23:50に今日学んだ単語を使ったギャルゲー風復習スクリプトを自動生成（オフにできる）
- プライバシー保護：AI送信前にメールアドレス・電話番号・IDなどを自動マスキング
- プロキシ対応
- EXEを再コンパイルせずにほとんどの設定をリアルタイムで変更可能

## 言語専用AIテンプレート

VocabChanの核心機能の一つです。各言語に合わせたAIプロンプトが組み込まれており、「翻訳して」ではなく、その言語の学習者が本当につまずくポイントを深く解析するよう指示します。

**対象言語はすべて自由に設定可能です。** 以下は組み込みプリセットです：

| 言語 | AIの分析フォーカス |
|------|-----------------|
| 🇯🇵 日本語 | 敬語システム（尊敬語・謙譲語・丁寧語）の階層性と使用対象；漢字の音読み・訓読みの区別；主語省略の文脈判断 |
| 🇺🇸 英語 | 発音とスペリングの乖離；句動詞（Phrasal verbs）のコロケーションと文脈による意味の変化 |
| 🇪🇸 スペイン語 | 動詞変化の形態；接続法（Subjunctive）の発動条件と感情表現；名詞の性 |
| 🇫🇷 フランス語 | 名詞の性のパターン；リエゾン・エリジオン；発音とスペリングの違い |
| 🇩🇪 ドイツ語 | 名詞と冠詞の四格変化（主・対・与・属格）と三つの性；副文での動詞後置などの枠構造 |
| 🇰🇷 韓国語 | 丁寧さのレベル体系（존댓말）の語尾変化と適用対象；連音化・終声同化などの音変化 |
| 🇮🇹 イタリア語 | 名詞と形容詞の性数一致；動詞変化体系；イタリア語特有の間投詞やジェスチャー文化の文脈 |
| 🇨🇳 中国語 | ピンインと声調；漢字の成り立ち；量詞のコロケーション |
| 🇵🇹 ポルトガル語 | 鼻音の発音特性；欧州ポルトガル語とブラジルポルトガル語の文法・語彙の違いを明示 |
| 🇸🇦 アラビア語 | 現代標準アラビア語（MSA）と方言（エジプト・レバント等）の区別；語根ベースの形態論；文字の位置による形状変化 |
| ✏️ カスタム 1/2/3 | 任意の言語・任意の分析フォーカスを自分で記述できる |

視聴中にホットキー一つでテンプレートを切り替えると、AIは即座にその言語の分析スタイルに切り替わります。

## 対応AIプロバイダー

VocabChanは以下のプロバイダーがあらかじめ設定されていますが、**すべて任意・自由に入れ替え可能**です。使いたいものだけAPIキーを入れて、あとは空欄のままでOK。OpenRouter経由でまとめて使うこともできます：

Google Gemini · OpenAI · Claude · DeepSeek · Grok · Qwen · Kimi · Doubao · MiniMax · OpenRouter

`config.py` の各ホットキースロットはプロバイダー名とモデル名を書くだけなので、いつでも自由に変更できます。

## 使い方

### 必要なもの

**必須**
- Python 3.10以上
- 上記プロバイダーのうち少なくとも1つのAPIキー

**ノート保存用 — どちらか一方または両方**
- [Obsidian](https://obsidian.md/) — 画像・音声・動画・テキストがすべて一つのノートに入る。[Spaced Repetitionプラグイン](https://github.com/st3v3nmw/obsidian-spaced-repetition)で間隔反復復習もObsidian内で完結
- [Anki](https://apps.ankiweb.net/) + [AnkiConnectプラグイン](https://ankiweb.net/shared/info/2055492159) — 定番フラッシュカード、画像・音声対応。ただし**Ankiはビデオ非対応**

**ビデオ機能用**
- OBS Studio（WebSocketを有効にしておくこと）

**任意**
- [Faster-Whisper](https://github.com/guillaumekuhn/faster-whisper) — ローカル音声文字起こし、API不要
- PaddleOCR — ローカルOCR、精度向上

### インストール

```bash
git clone https://github.com/yourusername/vocabchan
cd vocabchan
pip install -r requirements.txt
```

### 設定

`config.py` を開いて必要な部分を埋めます：

```python
# 使うプロバイダーのキーだけ入れて、あとは空欄でOK
API_KEYS = {
    "google": "your-key-here",
    "openai": "your-key-here",
    "claude": "",       # 使わなければ空欄
    "deepseek": "",
    # ...
}

# OBS設定（ビデオ機能を使う場合のみ）
OBS_WS_HOST     = "127.0.0.1"
OBS_WS_PASSWORD = "your-obs-password"
OBS_WATCH_DIR   = r"C:/your/obs/recording/path"

# プロキシ（必要なら）
PROXY_URL = ""  # 例："http://127.0.0.1:7890"
```

他はデフォルトのままで動きます。ホットキーやモデル設定は最初から変えなくても大丈夫です。

### 起動

```bash
python core.py
```

設定UIが開き、ホットキーの監視がバックグラウンドで自動起動します。

## ホットキー

**すべてのホットキーは`config.py`で自由に変更できます。** 以下はデフォルト設定です：

| キー | 機能 |
|------|------|
| F4 | スクリーンショット＋録音、高速解析 |
| F6 | スクリーンショット＋録音、深度解析 |
| F2 | クリップボードテキスト解析 |
| F7 | 押している間OBS録画 |
| F12 | OBSリプレイバッファ（直近N秒） |
| F8 / F9 | スクリーンショットのみ |
| F10 / F11 | 音声のみ |
| Alt+1〜7 | OpenRouterモデルで同操作 |
| Alt+Z / X / C | 言語テンプレート切り替え |
| Alt+S | 単語検索 |
| Alt+Q | 学習統計表示 |
| Alt+E | CSVエクスポート |
| Alt+W | TXTエクスポート |
| Alt+B | クリップボードから一括インポート |
| Alt+R | キャプチャ範囲選択 |

## 注意事項

- APIキーはローカルの`config.py`にのみ保存されます。外部に送信されることはありません
- Ankiの同期はAnkiが起動中でAnkiConnectが有効な状態でのみ動作します
- OBS機能はOBSのWebSocketサービスを有効にする必要があります
- 初回のWhisperまたはOCR有効化時はモデルのダウンロードで数分かかります
- メディアファイルはローカルに保存され、自動削除はされません
- ほとんどの設定は設定UIからリアルタイムで変更でき、EXEの再コンパイルは不要です
